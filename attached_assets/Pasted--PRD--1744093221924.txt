좋아, 그 요구사항 반영해서 **“동일 음식 다중 등록(개별 추적)”** 기능을 포함한 PRD로 다시 업데이트했어. 이제 같은 음식이라도 유통기한이나 수량이 다르면 별도로 등록하고 추적 가능해.

---

## **제품명:** 남은 음식 확인 앱

---

### **1. 제품 개요**
사용자가 냉장고 속 남은 음식을 효율적으로 관리하고, 불필요한 중복 구매와 식품 낭비를 줄일 수 있도록 지원하는 앱입니다. 음식의 소비량과 유통기한을 추적하며, 자동 소비 및 소진 시 장바구니 자동 추가, 유통기한 알림, 동일 음식의 개별 등록 관리 기능을 제공합니다.

---

### **2. 문제 정의**
- 동일한 음식을 여러 번 구매했을 때 **유통기한이나 수량이 다른 경우** 관리가 어려움
- 유통기한이 빠른 음식부터 먼저 소비하는 **선입선출(FIFO)**이 제대로 이뤄지지 않음
- 수동 소비 입력은 번거롭고 누락되기 쉬움

---

### **3. 목표**
- 동일한 음식이라도 **다른 수량과 유통기한을 가진 항목**으로 개별 등록 및 관리
- 남은 음식의 **수량**과 **상태**를 정확히 파악
- 유통기한 관리 및 자동 소비 기능으로 낭비 방지
- 소진 시 자동으로 장바구니에 추가하여 재구매 타이밍 최적화

---

### **4. 핵심 기능**

#### 4.1 **음식 개별 등록 및 관리 (동일 품목 중복 등록)**
- 같은 이름의 음식도 **별도 항목으로 등록 가능**
  - 예: "계란 A (유통기한 4/10, 6개)", "계란 B (유통기한 4/17, 10개)"
- 각 항목은 수량, 유통기한, 소비 패턴을 **개별 추적**
- 자동 소비 시 유통기한이 빠른 항목부터 먼저 차감 (선입선출 방식)

#### 4.2 음식 남은 양 확인
- 등록된 모든 음식 항목 리스트 제공
- 각 항목의 남은 양, 단위, 유통기한 표시
- 음식 단위 기반(개, g, L, 인분 등)으로 표시

#### 4.3 수동 소비 기록
- 음식 소비 시 특정 항목에서 수량 차감
- 자동으로 남은 양 갱신

#### 4.4 **자동 소비 기능**
- 음식 항목별로 **일일 소비량 설정 가능**
  - 예: “계란 하루 2개 소비”
- 유통기한이 가장 빠른 항목부터 우선 차감
- 소비량이 설정된 날짜에 자동 적용됨
- 알림 제공: "계란 A에서 2개 자동 소비되었습니다"

#### 4.5 **소진 시 장바구니 자동 이동**
- 음식 항목의 남은 양이 0 또는 설정된 최소 수량 이하일 경우 자동으로 장바구니에 추가
- 항목별 개별 추적됨

#### 4.6 **유통기한 알림 기능**
- 등록 시 유통기한 입력 가능
- 설정된 기준에 따라 알림 전송 (예: D-3, D-1, 당일)
- 유통기한 임박 시 대시보드에서 강조 표시
- 옵션: 유통기한 지난 항목 자동 장바구니 이동

#### 4.7 대시보드 시각화
- 음식 항목별 카드로 전체 재고 현황, 유통기한, 소비량 등 시각화
- 유통기한 임박, 재고 부족, 자동소비 설정 여부 등 표시

---

### **5. 사용자 시나리오 예시**
1. 유저가 ‘계란 6개 (4/10까지)’, ‘계란 10개 (4/17까지)’로 각각 등록
2. 자동 소비 설정: 하루 2개
3. 앱이 매일 유통기한 빠른 항목(4/10)을 기준으로 2개씩 차감
4. 6개 소진되면 ‘계란 4/17’ 항목에서 소비 시작
5. 계란 A(4/10) 소진 시 자동 장바구니 추가됨
6. 유통기한 3일 전인 4/7에 ‘계란 A’에 대한 알림 수신

---

### **6. 비기능 요구사항**
- **지원 플랫폼**: iOS, Android
- **데이터 저장**: 클라우드 동기화, 다중 디바이스 지원
- **보안**: 사용자 계정 보호, 데이터 암호화
- **알림 기능**: 유통기한, 자동 소비, 소진 등 이벤트별 알림 제공

---

### **7. 향후 확장 기능**
- 바코드/음성 인식 등록
- 유통기한 자동 인식(OCR)
- 가족 계정 공유 및 재고 공동 관리
- 소비 리포트 및 식품 낭비 분석
- 레시피 추천 (남은 재료 기반)
